JavaScript in Web Development

07/13/2014

  • History

    • Javascript was developped by Brendan Eich in the 90s out of a need to make webpages dynamic.

  • Why has Javascript has gained so much popularity in recent times?

    1. It's easy to learn.
    2. Javascript code is executed on a user's processor instead of a web server thus saving the user the time and bandwidth that would have been consumed had all the processing been done on the web server.
    3. Javascript is a prototype language. This means that:
      • objects can be defined and developed on the fly rather than having to build specific classes to meet specific needs.
      • you can easily modify or extend the functionality of existing classes.
    4. It's easy to debug using the browser's console or in Firebug.
    5. It has a lot of tools and is easily extensible.

  • Drawbacks

    • Security
        Since javascript code is executed on the client's processor, it's easy for malicious code to reach end users and exploit their systems.
    • Rendering
        Javascript could be interpreted differently by different browsers and thus produce different outputs. However, this can be avoided by testing how JavaScript renders on different browsers.
