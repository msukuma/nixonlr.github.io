SQL vs noSQL

08/10/2014

What's the difference between SQL and noSQL?

The major difference between these database management systems (DBMS) is in how they represent data. SQL databases do so in the form of tables and adhere to a schema definition. NoSQL databases, on the other hand, are more flexible and can comprise of key-value pair, documents, graph databases, wide-column stores etc...

NoSQL databases scale up horizontally because of auto-sharding which allows NoSQL systems to automatically share data across servers thereby balancing the load across multiple servers. However, SQL databases scale up vertically, adding more and more traffic to a single server.

So SQL or noSQL?

Which type of database system to use depends on your needs and expectations. You should ask yourself the following before deciding:

  • How big a company are you?
  • How many relationships and tables are going to make up your SQL database?
  • How many lookups do you anticipate users will make daily?
  • What data types will you be using?

If you are big company like Google who uses complex data, and whose databases relationships and lookups range in the millions, then a noSQL DBMS will probably be your best choice.
